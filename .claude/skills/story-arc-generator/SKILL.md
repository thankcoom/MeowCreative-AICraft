## 功能說明

此 Skill 提供故事弧線生成功能，幫助將文章轉化為引人入勝的敘事結構。

### 主要功能

1. **故事結構分析** - 分析現有文章的敘事結構
2. **弧線生成** - 根據主題生成適合的故事弧線
3. **情感曲線設計** - 設計讀者情感起伏路徑
4. **轉折點建議** - 提供關鍵轉折點設計
5. **角色旅程規劃** - 設計主角（讀者/用戶）的轉變旅程

---

## 使用方式

### 命令列使用

```bash
# 分析現有文章的故事結構
python3 .claude/skills/story-arc-generator/generate.py \
    input_article.md \
    --mode analyze \
    --output story_analysis.md

# 生成故事弧線建議
python3 .claude/skills/story-arc-generator/generate.py \
    input_article.md \
    --mode generate \
    --structure hero_journey \
    --output story_arc.md

# 生成情感曲線
python3 .claude/skills/story-arc-generator/generate.py \
    input_article.md \
    --mode emotion \
    --pattern man_in_hole \
    --output emotion_curve.md
```

### 參數說明

| 參數 | 必需 | 說明 |
|------|------|------|
| `input` | 是 | 待分析的文章路徑或主題描述 |
| `--mode`, `-m` | 否 | 模式: `analyze`, `generate`, `emotion`（預設: analyze） |
| `--structure`, `-s` | 否 | 故事結構: `hero_journey`, `three_act`, `story_spine`, `freytag`（預設: hero_journey） |
| `--pattern`, `-p` | 否 | 情感模式: `rags_to_riches`, `man_in_hole`, `cinderella`, `icarus`（預設: man_in_hole） |
| `--output`, `-o` | 否 | 輸出路徑（預設: story_arc.md） |
| `--format` | 否 | 輸出格式: `markdown`, `json`（預設: markdown） |

## 故事結構模板

### 1. Hero's Journey（英雄之旅）

適合：轉型故事、學習歷程、個人成長

| 階段 | 名稱 | 文章應用 |
|------|------|----------|
| 1 | 平凡世界 | 讀者現狀描述 |
| 2 | 冒險召喚 | 問題/機會出現 |
| 3 | 拒絕召喚 | 常見顧慮和阻力 |
| 4 | 遇見導師 | 解決方案/工具介紹 |
| 5 | 跨越門檻 | 開始行動 |
| 6 | 試煉盟友敵人 | 執行過程中的挑戰 |
| 7 | 逼近洞穴 | 最困難的部分 |
| 8 | 苦難折磨 | 關鍵突破點 |
| 9 | 獲得寶物 | 掌握核心技能 |
| 10 | 歸途 | 整合應用 |
| 11 | 復活 | 持續精進 |
| 12 | 帶著寶物歸來 | 成果和價值 |

### 2. Three-Act Structure（三幕劇）

適合：問題解決、教學文章

| 幕 | 比例 | 內容 |
|----|------|------|
| 第一幕：設定 | 25% | 現狀、問題、利害關係 |
| 第二幕：對抗 | 50% | 嘗試、失敗、學習、突破 |
| 第三幕：解決 | 25% | 解決方案、成果、未來展望 |

### 3. Story Spine（Pixar 故事骨架）

適合：案例分享、經驗談

```
從前有... (背景設定)
每天... (日常狀態)
直到有一天... (觸發事件)
因為這樣... (後果1)
因為這樣... (後果2)
因為這樣... (後果3)
直到最後... (高潮)
從那之後... (新常態)
```

### 4. Freytag's Pyramid（弗萊塔格金字塔）

適合：戲劇性內容、懸念文章

```
        高潮
       /    \
  上升/      \下降
     /        \
開端 ──────── 結局
```

## 輸出格式

### 分析模式輸出

```markdown
# 故事結構分析報告

## 現有結構檢測

| 元素 | 狀態 | 位置 | 說明 |
|------|------|------|------|
| 開場鉤子 | ✅ | 第1段 | 使用問題引入 |
| 衝突設定 | ⚠️ | 缺失 | 建議增加 |
| 轉折點 | ✅ | 第5段 | 發現解決方案 |
| 高潮 | ❌ | 缺失 | 需要添加 |
| 結局 | ✅ | 最後段 | 總結性結尾 |

## 敘事強度評分

- **整體分數**: 62/100
- **開頭吸引力**: 75/100
- **中段張力**: 55/100
- **結尾收束**: 60/100

## 建議結構

建議採用：Hero's Journey（簡化版）
原因：文章主題適合轉型敘事
```

### 生成模式輸出

```markdown
# 故事弧線設計

## 推薦結構：Hero's Journey

### 段落規劃

| 段落 | 階段 | 內容建議 | 字數 |
|------|------|----------|------|
| 1 | 平凡世界 | 描述讀者目前的痛點 | 150 |
| 2 | 冒險召喚 | 提出新的可能性 | 100 |
| 3 | 遇見導師 | 介紹解決方案 | 200 |
| 4-6 | 試煉 | 步驟說明和挑戰 | 600 |
| 7 | 苦難折磨 | 最難的部分和突破 | 150 |
| 8-9 | 獲得寶物 | 成果展示 | 200 |
| 10 | 歸來 | 總結和行動呼籲 | 100 |

### 轉折點設計

1. **第一轉折** (約20%處): 從問題到解決方案
2. **中點** (約50%處): 最大挑戰出現
3. **第二轉折** (約80%處): 突破達成

### 情感曲線

採用「Man in a Hole」模式：
- 開頭：讀者共鳴（中性）
- 問題呈現：情感下降
- 最低點：最大痛點
- 解決方案：情感回升
- 結尾：高於開頭的正面情感
```

## 評分標準

### 敘事強度評分

| 分數 | 等級 | 說明 |
|------|------|------|
| 90-100 | A+ | 卓越，故事性極強 |
| 80-89 | A | 優秀，引人入勝 |
| 70-79 | B | 良好，有基本敘事 |
| 60-69 | C | 及格，敘事較弱 |
| <60 | D | 需改進，缺乏故事性 |

### 通過標準

- 敘事強度 ≥ 70/100
- 必須有明確的開頭、中段、結尾
- 至少包含一個轉折點
- 情感曲線有起伏
