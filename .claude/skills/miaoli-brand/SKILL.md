# 喵哩文創品牌風格指南

> **品牌資料**: 完整的品牌 Profile 儲存在 `.claude/memory/brands/miaoli.yaml`
>
> 使用此 Skill 時，請先讀取品牌資料以獲取最新設定。

## 概述

這個 skill 確保所有創作內容符合喵哩文創的品牌形象、語調和格式規範。

**適用場景**: 文章撰寫、社群貼文、行銷素材、電子報、任何需要保持品牌一致性的內容

## 品牌核心價值

### 品牌定位
- **真實性**: 基於真實經驗，拒絕虛構內容
- **專業性**: 提供有價值的洞察和實用建議
- **親和力**: 對話式語調，易於理解
- **創新性**: 擁抱 AI 工具，持續優化流程

### 目標受眾
- **主要**: 中文部落客、內容創作者、數位行銷人員
- **次要**: 小型企業主、自由工作者、學生
- **年齡層**: 25-45 歲
- **需求**: 高效創作、SEO 優化、內容行銷

## 語調與風格

### 語調原則

#### ✅ 應該使用
- **對話式**: 使用「我」「你」，避免「本文」「讀者」
- **親和但專業**: 友善但不失專業度
- **實用導向**: 重視可操作的建議
- **真實誠懇**: 坦承限制，不誇大效果

**範例**:
```
✅ 好: "我在測試這個工具時發現了三個實用技巧，可以幫你節省至少 30% 的時間。"
❌ 差: "本研究發現該工具具有顯著的效率提升效果。"
```

#### ❌ 應該避免
- 過度正式或學術化
- 過度行銷或誇大
- 冷冰冰的專業術語堆砌
- 虛構的個人經驗或數據

### 句式結構

- **句長**: 平均 15-20 字，最長不超過 30 字
- **段落**: 每段 3-5 句話
- **標題**: 清晰、具體、包含關鍵字

**範例**:
```markdown
## 如何在 10 分鐘內優化文章 SEO

我最近發現一個快速優化 SEO 的方法。這個方法不需要任何付費工具，只需要三個步驟。

第一步是檢查關鍵字密度。你可以用 Ctrl+F 搜尋主要關鍵字，確保它在標題、首段和小標題中都有出現。理想的密度是 1-2%，太多會被視為關鍵字堆砌。

第二步是優化 Meta Description。這是搜尋結果中顯示的描述文字，長度應該控制在 150-160 字。記得在描述中加入主要關鍵字和行動呼籲。

第三步是檢查內部連結。每 300 字應該有 1 個相關文章的連結，但不要過多。這能幫助讀者發現更多內容，也能提升 SEO 分數。
```

## 內容結構規範

### 文章結構

1. **標題 (H1)**
   - 只能有 1 個 H1
   - 長度: 20-40 字
   - 必須包含主要關鍵字
   - 使用數字或疑問句增加吸引力

2. **導言 (首段)**
   - 長度: 80-150 字
   - 說明文章主題和價值
   - 包含主要關鍵字
   - 可以使用「你是否...」開頭引起共鳴

3. **主體 (H2-H3)**
   - H2: 3-5 個主要章節
   - H3: 適量，依內容需求
   - 每個 H2 下至少 2-3 段內容
   - 每個段落 3-5 句話

4. **結論**
   - 總結主要觀點 (3-5 點)
   - 提供行動建議
   - 可選: 提問引發討論

5. **CTA (行動呼籲)**
   - 簡短有力
   - 明確的下一步
   - 可選: 訂閱、下載、評論

### 視覺元素

- **清單**: 優先使用項目符號或數字清單
- **粗體**: 強調重點詞彙（但不要過度使用）
- **程式碼**: 技術內容使用程式碼區塊
- **引用**: 引用數據或他人觀點使用 blockquote
- **表格**: 比較性內容使用表格
- **圖片**: 每 500-800 字搭配 1 張相關圖片（如可獲取）

## SEO 規範

### 關鍵字使用

- **主要關鍵字** (1-2 個):
  - 出現在: H1、首段、至少 2 個 H2、結論
  - 密度: 1-2%
  - 自然融入，不生硬堆砌

- **次要關鍵字** (3-5 個):
  - 出現在: H2、H3、內文
  - 密度: 0.5-1%

- **LSI 關鍵字** (5-10 個):
  - 相關詞彙和同義詞
  - 自然分布在全文

### Meta 資訊

- **Meta Title**: 50-60 字，包含主要關鍵字
- **Meta Description**: 150-160 字，包含關鍵字和 CTA
- **URL Slug**: 3-5 個字，全小寫，用 `-` 分隔，包含關鍵字

### 連結策略

- **內部連結**: 每 300 字 ≤ 1 個相關文章連結
- **外部連結**: 3-5 個權威來源（官方文件、研究報告、知名媒體）
- **錨文本**: 使用描述性文字，避免「點這裡」

## 引用與來源

### 引用規範

所有數據、統計、專業觀點都必須標註來源：

```markdown
根據 Google 的研究報告 [1]，優質內容的平均字數為 1,890 字。

[1] https://example.com/google-research-2024
```

### 真實性原則

- **個人經驗**: 必須來自 `experience_profile.md`
- **案例研究**: 必須可驗證或明確標註為範例
- **數據統計**: 必須引用可靠來源
- **產品評測**: 必須實際使用過或明確說明

## 不同平台的調整

### WordPress 部落格

- 字數: 1,500-3,000 字
- 語調: 親和但專業
- SEO: 完整優化
- 結構: 完整的 H1-H3 階層

### Twitter/X

- 字數: 250-280 字
- 語調: 輕鬆、有趣
- Hook: 前 50 字最重要
- Hashtags: 2-3 個相關標籤

### LinkedIn

- 字數: 300-500 字
- 語調: 專業、價值導向
- 開頭: 問句或統計數據
- 結尾: 明確的討論引導

### Facebook

- 字數: 300-500 字
- 語調: 親和、引發討論
- 視覺: 配圖重要
- 互動: 提問結尾

### Email / 電子報

- 字數: 500-800 字
- 語調: 個人化、親切
- 主旨: 50 字內，引發好奇
- CTA: 明確且單一

## 品質檢查清單

在發布前，確認以下項目：

### 內容品質
- [ ] 所有數據都有來源
- [ ] 沒有虛構的個人經驗
- [ ] 語調一致（對話式、親和）
- [ ] 句子簡潔（平均 15-20 字）
- [ ] 段落適中（3-5 句話）

### SEO 優化
- [ ] H1 只有 1 個
- [ ] 主要關鍵字密度 1-2%
- [ ] Meta Description 150-160 字
- [ ] 內部連結適量（每 300 字 ≤ 1 個）
- [ ] 外部引用 3-5 個權威來源

### 格式規範
- [ ] 標題階層正確（H1 → H2 → H3）
- [ ] 使用清單增強可讀性
- [ ] 重點詞彙使用粗體（適度）
- [ ] 程式碼使用程式碼區塊
- [ ] 適當使用視覺元素

### 品牌一致性
- [ ] 語調符合品牌定位
- [ ] 專業術語統一
- [ ] 符合目標受眾需求
- [ ] 體現品牌核心價值
